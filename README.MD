# Room Booking Service

Сервис бронирования переговорок для компаний. Позволяет пользователям просматривать доступные переговорки и бронировать временные слоты, а администраторам — управлять расписаниями и переговорками.

## Основные возможности

- **Управление переговорками**: создание и просмотр списка переговорок (только для администраторов)
- **Расписание**: создание расписаний с автоматической генерацией слотов на 7 дней вперёд
- **Бронирование**: бронирование свободных слотов пользователями с защитой от двойного бронирования
- **Авторизация**: JWT-аутентификация с разграничением ролей (admin/user)
- **Документация**: Swagger UI для тестирования API

## Стек технологий

- **Язык**: Go 1.24
- **База данных**: PostgreSQL 16
- **ORM**: GORM
- **Аутентификация**: JWT (golang-jwt/jwt/v5)
- **Контейнеризация**: Docker Compose
- **Документация**: Swagger (swaggo)

## Быстрый старт


## API Endpoints

### Аутентификация

| Метод | Путь | Роль | Описание |
|-------|------|------|----------|
| POST | /dummyLogin | - | Получить тестовый JWT (для разработки) |
| POST | /register | - | Регистрация нового пользователя |
| POST | /login | - | Авторизация по email/паролю |

### Переговорки

| Метод | Путь | Роль | Описание |
|-------|------|------|----------|
| POST | /rooms/create | admin | Создать новую переговорку |
| GET | /rooms/list | admin/user | Получить список всех переговорок |

### Расписание и слоты

| Метод | Путь | Роль | Описание |
|-------|------|------|----------|
| POST | /rooms/{roomId}/schedule/create | admin | Создать расписание для переговорки |
| GET | /rooms/{roomId}/slots/list?date=YYYY-MM-DD | admin/user | Получить свободные слоты на дату |

### Бронирования

| Метод | Путь | Роль | Описание |
|-------|------|------|----------|
| POST | /bookings/create | user | Создать бронирование |
| GET | /bookings/my | user | Получить список своих бронирований |
| POST | /bookings/{bookingId}/cancel | user | Отменить бронирование |
| GET | /bookings/list | admin | Получить все бронирования |

### Системные

| Метод | Путь | Описание |
|-------|------|----------|
| GET | /_info | Healthcheck |

## Архитектурные решения

### Генерация слотов
Слоты генерируются при создании расписания на ближайшие 7 дней вперёд. Этот подход выбран из-за того, что 99.9% запросов приходятся на ближайшую неделю — генерация при каждом запросе была бы избыточной.

### Фильтрация занятых слотов
Эндпоинт `GET /rooms/{roomId}/slots/list` возвращает только свободные слоты — те, которые не имеют активной брони. Фильтрация происходит в `schedule/service.go`.

### Конкурентное бронирование
Защита от двойного бронирования реализована через partial unique index в БД:
```sql
CREATE UNIQUE INDEX idx_active_booking_slot ON bookings (slot_id) WHERE status = 'active'
```
База данных гарантирует, что только одна активная бронь может существовать для слота.

### Идемпотентность отмены
Повторная отмена уже отменённой брони возвращает 200 с актуальным состоянием брони без ошибки.

### Авторизация
Используется JWT с полями `user_id` и `role`. Эндпоинт `/dummyLogin` возвращает фиксированные UUID для каждой роли:
- admin: `00000000-0000-0000-0000-000000000001`
- user: `00000000-0000-0000-0000-000000000002`
